## 引言

小程序开发完成后需经过调试、审核、发布流程。开发者工具提供模拟器、调试器、网络面板等，可完成大部分开发调试；但支付、定位、部分 API 等需真机验证。发布前需上传代码、提交审核、通过后发布，用户才能搜索或通过链接进入。

本文介绍开发者工具调试技巧、真机预览、体验版与正式版发布流程。建议在体验版充分测试后再提交审核，减少驳回次数。

## 开发者工具调试

开发者工具提供 Console（日志、错误）、Network（请求详情）、Storage（本地存储）、AppData（页面 data 实时查看）等面板。性能面板可分析 setData 耗时、渲染耗时。调试时可使用 `console.log`、断点、`vConsole` 等。

- Console 查看日志与错误
- Network 面板查看请求
- 调试器 - AppData 查看 data 状态
- 性能面板分析 setData、渲染耗时

## 真机预览

点击「预览」生成二维码，手机扫码体验。注意真机与模拟器差异：支付、定位、部分 API 等需真机验证。

## 发布流程

1. 上传代码：点击「上传」，填写版本号与备注
2. 登录公众平台，提交审核：选择版本、填写类目与说明
3. 审核通过后，在「版本管理」中发布
4. 发布后用户可搜索或通过链接进入

## 总结

调试阶段充分利用模拟器与真机，发布前检查配置与合规（如隐私协议、类目、服务内容等）。熟悉审核规范，可减少驳回次数，加快上线节奏。体验版可供内部或指定用户测试，正式版对所有用户开放。
